Blueprint Aplikasi SIPUSAKA

Deskripsi arsitektur aplikasi perpustakaan berbasis Laravel, dengan alur publik dan admin, model, layanan denda, rute, dan halaman utama.

1. Tujuan Aplikasi

Aplikasi ini menyediakan sistem peminjaman buku perpustakaan, meliputi:

  • Halaman publik untuk melihat katalog, meminjam buku, cek status peminjaman, dan QR peminjaman.
  • Panel admin untuk mengelola buku, mahasiswa, peminjaman, pengembalian, denda, dan riwayat.
  • Perhitungan denda otomatis untuk keterlambatan pengembalian.

2. Struktur Model Utama

Models

  • Buku - data buku perpustakaan
  • Mahasiswa - data peminjam mahasiswa
  • Peminjaman - transaksi peminjaman buku
  • Denda - tagihan keterlambatan
  • History - catatan riwayat aksi admin
  • User - akun administrator

Relasi Antar Model

ModelRelasiTarget
PeminjamanbelongsToMahasiswa
PeminjamanbelongsToBuku
PeminjamanhasOneDenda
DendabelongsToPeminjaman
HistorybelongsToPeminjaman

3. Alur Publik

Publik menggunakan controller App\Http\Controllers\Publik\PinjamController dan App\Http\Controllers\Publik\KatalogController.
  • Halaman Katalog - /, /buku/{id}, /katalog/search.
  • Form Peminjaman - /pinjam untuk mengajukan peminjaman.
  • Cek NIM - /pinjam/cek-nim untuk verifikasi mahasiswa.
  • QR Peminjaman - /pinjam/get-qr untuk mendapatkan data dari QR.
  • Konfirmasi - /pinjam/konfirmasi/{booking_id}.
  • Cek Status - /cek-status dan /cek-status (POST) menampilkan status peminjaman.

5. Layanan Denda

Service App\Services\DendaService menghitung denda dari perbedaan tanggal:

hari_terlambat = max(0, diffInDays(tanggal_kembali_aktual, tanggal_kembali_rencana))
total_denda = hari_terlambat * 10000

Layanan ini dipakai ketika admin memproses pengembalian.

7. Database & Skema

  • pinjamans - kolom penting: mahasiswa_id, buku_id, tanggal_pinjam, tanggal_kembali_rencana, tanggal_kembali_aktual, status, booking_id.
  • dendas - kolom penting: pinjaman_id, hari_terlambat, total_denda, status_bayar, dibayar_at.
  • bukus - data buku dan stok.
  • mahasiswas - data identitas mahasiswa.
  • histories - riwayat aksi admin terhadap peminjaman.

8. Layout & Struktur Tampilan

Layout utama menggunakan folder:

  • resources/views/layouts/admin.blade.php
  • resources/views/layouts/publik.blade.php

Semua halaman publik dan admin mewarisi layout ini untuk konsistensi UI.

9. Catatan Implementasi

  • Admin dapat memproses pengembalian lewat scan QR atau input booking ID.
  • Jika keterlambatan terjadi, denda otomatis dibuat atau diperbarui.
  • Halaman publik menampilkan status peminjaman dan detail denda jika ada.
  • Fitur export tersedia untuk buku, peminjaman, denda, dan history.
